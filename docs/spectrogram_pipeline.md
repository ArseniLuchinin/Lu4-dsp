# Конвейер построения спектрограммы

## Обзор

Конвейер спектрограммы демонстрирует полный цикл спектрального анализа сигнала на GPU: от чтения комплексных IQ-отсчётов из файла до готового PNG-изображения. Вся обработка выполняется без промежуточного переноса данных на CPU — все модули работают с GPU-контейнерами (`GpuSignal`), что минимизирует накладные расходы на копирование и синхронизацию.

Конфигурация конвейера находится в файле `SpectrogramPipeline.json`.

---

## Состав конвейера

```
┌─────────┐   GpuComplexFloatSignal   ┌─────┐   GpuComplexFloatSignal   ┌────────┐
│ FileSrc │ ────────────────────────→ │ FFT │ ────────────────────────→ │ CS2AS  │
└─────────┘                           └─────┘                           └───┬────┘
                                                                            │
                              GpuFloatSignal (амплитудный спектр, дБ)      │
                                                                            ▼
┌─────────────────┐   GpuFloatSignal   ┌───────────┐   GpuFloatSignal   ┌───────────────┐
│ SpectrogramPlot │ ←───────────────── │ FFT_Shift │ ←───────────────── │  (из CS2AS)   │
└─────────────────┘                    └───────────┘                    └───────────────┘
```

| # | Модуль | Назначение |
|---|--------|------------|
| 1 | `FileSrc` | Чтение IQ-сигнала из бинарного файла напрямую на GPU |
| 2 | `FFT` | Быстрое преобразование Фурье через cuFFT с overlap-add |
| 3 | `CS2AS` | Перевод комплексного спектра в амплитудный (дБ) |
| 4 | `FFT_Shift` | Центрирование нулевой частоты (in-place) |
| 5 | `SpectrogramPlot` | GPU-рендеринг цветовой карты и сохранение PNG |

## Поток данных

### 1. FileSrc — источник сигнала

Модуль отображает файл в память (`mmap`) и копирует блоки заданного размера на GPU через `cudaMemcpy`. В случае спектрограммы используется комплексный формат (`data type = complex`), поэтому на выходе формируется `GpuComplexFloatSignal`.

**Ключевые параметры:**
| Параметр | Значение | Описание |
|----------|----------|----------|
| `file name` | `$SIGNAL_PATH` (`noised_amc_signal.bin`) | Путь к входному файлу |
| `data type` | `complex` | Формат отсчётов |
| `max size` | `$MAX_SIZE` (268 435 456 байт) | Максимальный размер одного блока |

> **Оптимизация.** `mmap` позволяет избежать лишнего копирования в пользовательском буфере при чтении с диска. Данные сразу загружаются в GPU-память, не задерживаясь на CPU.

### 2. FFT — спектральное преобразование

Реализовано через NVIDIA cuFFT. Модуль автоматически определяет тип входных данных (вещественный или комплексный) и формирует соответствующий план `cufftPlanMany`.

**Overlap-add через callbacks**

Для потоковой обработки с перекрытием используется механизм `cufftXtSetCallback`: вместо явного формирования окон с шагом `hop size` callback-функции `overlap_load_cb` и `overlap_load_cb_real` выполняют загрузку с учётом overlap непосредственно внутри cuFFT. Это устраняет дополнительный этап копирования и перетасовки данных в промежуточном буфере.

**Ключевые параметры:**
| Параметр | Значение | Описание |
|----------|----------|----------|
| `fft size` | `$FFT_SIZE` (16 384) | Размер окна FFT |
| `hop size` | `$FFT_HOP_SIZE` (8 192) | Шаг между соседними окнами (50 % перекрытия) |

**Вход:** `GpuComplexFloatSignal`  
**Выход:** `GpuComplexFloatSignal` — спектр для каждого окна

### 3. CS2AS — амплитудный спектр в децибелах

Модуль **Complex Spectrum → Amplitude Spectrum** переводит комплексные бины в логарифмическую амплитуду:

```
P = |I + jQ|² × norm
A[dB] = 10 × log10(P)
```

При `normalize by fft size = true` используется `norm = 1 / fftSize²`, что компенсирует энергетический выигрыш FFT и делает уровень независимым от размера окна.

Вычисление выполняется CUDA-ядром `complexToAmplitudeKernel` (inline в `.cpp`); каждый поток обрабатывает один бин, обеспечивая полное параллельное покрытие данных.

**Ключевые параметры:**
| Параметр | Значение | Описание |
|----------|----------|----------|
| `fft size` | `$FFT_SIZE` (16 384) | Размер FFT |
| `normalize by fft size` | `true` | Нормализация на квадрат размера FFT |

**Вход:** `GpuComplexFloatSignal`  
**Выход:** `GpuFloatSignal` (значения в дБ)

### 4. FFT_Shift — центрирование спектра

Меняет местами левую и правую половины каждой спектральной строки, чтобы нулевая частота оказалась в центре изображения. Реализовано in-place CUDA-ядром `fftShiftInplaceKernel`: каждый поток обменивает пару элементов, не требуя дополнительного выделения памяти.

> **Оптимизация.** In-place обработка исключает лишнее `cudaMalloc` / `cudaFree` и сокращает трафик памяти GPU вдвое.

**Ключевые параметры:**
| Параметр | Значение | Описание |
|----------|----------|----------|
| `fft size` | `$FFT_SIZE` (16 384) | Размер FFT |

**Вход:** `GpuFloatSignal`  
**Выход:** тот же `GpuFloatSignal` (in-place)

### 5. SpectrogramPlot — финальная визуализация

Модуль принимает амплитудный спектр и формирует из него полноценное изображение спектрограммы:

1. **GPU-рендеринг** — ядро `renderSpectrogramKernel` преобразует каждое значение дБ в цвет по палитре **Parula** и записывает результат в BGR-изображение в GPU-памяти.
2. **Масштабирование и подписи** — на CPU добавляются оси (частота / время) и легенда.
3. **Сохранение** — OpenCV записывает результат в PNG с заданным уровнем компрессии.

**Ключевые параметры:**
| Параметр | Значение | Описание |
|----------|----------|----------|
| `sample rate` | `$SAMPLE_RATE` (2 097 152 Гц) | Частота дискретизации для расчёта оси частот |
| `fft size` | `$FFT_SIZE` (16 384) | Размер FFT |
| `hop size` | `$FFT_HOP_SIZE` (8 192) | Шаг между окнами |
| `centered spectrum` | `true` | Спектр уже центрирован FFT_Shift |
| `db min` / `db max` | -60 / 0 дБ | Динамический диапазон цветовой шкалы |
| `mask below db` | -20 дБ | Значения ниже порога отображаются как прозрачные/фон |
| `show` | `false` | Не открывать окно просмотра |
| `save path` | `$SPECTROGRAM_PATH` | Префикс пути для выходных файлов |
| `png compression` | 1 | Уровень сжатия PNG (1 = быстрое, максимальный размер) |

> **Оптимизация.** Рендеринг цветовой карты полностью выполняется на GPU; хост получает уже готовое BGR-изображение, что существенно снижает нагрузку на CPU при построении больших спектрограмм. Так же выходное изображение можно обрезать, оставив только нужную полосу частот, это снижает нагрузку на CPU часть и занчительно ускоряет работу.

## CUDA-оптимизации в конвейере

| Оптимизация | Модуль | Эффект |
|-------------|--------|--------|
| **Zero-copy pipeline** | Все | Данные остаются на GPU на всём протяжении цепочки; между модулями передаётся только `shared_ptr<IData>` |
| **Overlap callbacks (cuFFT)** | FFT | Устраняет явное копирование перекрывающихся окон; overlap выполняется внутри cuFFT при загрузке |
| **In-place FFT_Shift** | FFT_Shift | Не требует дополнительного GPU-буфера |
| **GPU-рендеринг цвета** | SpectrogramPlot | Параллельное преобразование dB → RGB для всех пикселей одновременно |
| **Buffer reuse (`reserve`)** | Все | `GpuSignal::reserve()` не перевыделяет память, если новый размер ≤ текущей ёмкости |
| **Page-locked чтение** | FileSrc | `mmap` + прямое H2D-копирование минимизирует задержки на ввод |
