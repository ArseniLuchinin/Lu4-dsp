# AGENTS.md — Инструкции для AI-агентов

## Роль агента

Агент — это AI-помощник для работы с проектом **computing_server** (CUDA-ускоренный сервер обработки цифровых сигналов). Агент помогает с:
- Разработкой новых модулей
- Исправлением багов
- Рефакторингом кода
- Написанием тестов
- Анализом архитектуры

---

## Доступные инструменты

| Инструмент | Назначение |
|---|---|
| `read` | Чтение файлов (с ограничением по строкам) |
| `glob` | Поиск файлов по паттерну |
| `grep` | Поиск по содержимому (regex) |
| `edit` | Точечное редактирование (oldString → newString) |
| `write` | Перезапись файла |
| `bash` | Выполнение команд ОС |
| `question` | Запрос уточнений у пользователя |
| `task` | Запускsub-agent для исследования |
| `webfetch` / `websearch` / `codesearch` | Работа с вебом |

## Рабочие конвенции

### Перед началом работы
1. **Прочитать QWEN.md** — базовая документация проекта
2. **Определить область изменений** — какие файлы затрагиваются
3. **Найти релевантный код** — использовать grep/glob

### При редактировании кода
1. **Сохранять стиль** — смотреть на существующий код в файле
2. **Не добавлять комментарии** — без явного запроса пользователя
3. **Следовать именованию** — из QWEN.md (PascalCase, camelCase, m_ prefix)
4. **Проверять include order** — проект, внешние библиотеки, stdlib

### При создании нового кода
- C++17, без исключений
- std::shared_ptr для данных
- Boolean returns: true = успех, false = ошибка
- Boost.Log для логирования
- Doxygen-комментарии для интерфейсных методов

## Типичные задачи

### Добавление модуля
1. Создать директорию `Modules/<Name>/`
2. Добавить `include/`, `src/`, `CMakeLists.txt`
3. Реализовать интерфейс IModule
4. Экспортировать factory `createModule()`
5. Зарегистрировать в `Modules/module.hpp`
6. Добавить тесты в `tests/`
7. Создать README.md с документацией

### Исправление бага
1. Найти код через grep (по сообщению об ошибке)
2. Понять логику существующего кода
3. Внести исправление
4. Проверить тестами: `ctest --preset all -R <TestPattern> -V`
5. Запустить линтер/форматтер (если есть в CMakePresets.json)

### Рефакторинг
1. Найти все использования через grep
2. Обновить все файлы
3. Проверить, что тесты проходят

### Написание тестов
1. Добавить в `Modules/<Name>/tests/` или `tests/`
2. Следовать стилю: `TEST(TestSuite, TestName)`
3. Использовать ASSERT_TRUE, EXPECT_EQ

## Тести��ование

### Запуск тестов
```bash
ctest --preset all                    # все тесты
ctest --preset module                 # только модульные
ctest --preset e2e                   # только e2e
ctest --preset all -R "Pattern" -V    # конкретный тест
```

### Пропуск тестов
```cpp
GTEST_SKIP() << "Reason";  // если нет CUDA или другие причины
```

### Первый запуск (если тесты выключены)
```bash
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Debug -DBUILD_TESTING=ON
cmake --build build -j 6
```

## Сборка

### Конфигурация и сборка
```bash
cmake --preset debug
cmake --build --preset debug -j 6
```

### Запуск сервера
```bash

# По умолчанию: pipeline.json в корне проекта
```

## Ограничения

- **Не создавать коммиты** — без явного запроса пользователя
- **Не изменять .env / credentials** — не-committable файлы
- **Не использовать сырые указатели** — только для CUDA device memory
- **Не использовать std::unique_ptr** — только std::shared_ptr
- **Без исключений** — в коде модулей
